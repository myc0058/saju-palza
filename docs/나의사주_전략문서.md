## 1. 서비스 개요 및 핵심 아이디어

### 기존 사주 서비스의 문제

- **1회성 소비**: 운세 확인 후 재방문 동기 없음 → 고객 유지율 극히 낮음
- **통계적 평균 해석**: 같은 날 태어난 모두에게 동일한 내용 제공
- **부정확한 입력값**: 출생 시각 불명확으로 근본적 오류 발생
- **검증 구조 부재**: 예측이 맞았는지 틀렸는지 확인·보정할 수 있는 구조 없음

### 핵심 아이디어

기존 사주는 **생년월일시(고정값)** 만으로 해석한다. 같은 날 태어난 사람도 전혀 다른 삶을 산다. 이 서비스의 차별점은 다음 한 문장으로 요약된다.

> **"당신이 실제로 겪은 인생 데이터로 사주를 보정(calibration)한다"**

## 2. 동작 원리

### 전체 흐름

```
[1] 기본 사주 생성
       ↓
[2] 유저 인생 이벤트 수집
       ↓
[3] LLM → 이벤트 수치화 (-1.0 ~ 1.0)
       ↓
[4] 사주 변수에 가중치 보정
       ↓
[5] 개인화된 운세 해석 출력
       ↓
[6] 피드백 루프 (지속 정밀화)
```

### 1단계 — 기본 사주 생성 (정적 데이터)

생년월일시로 사주팔자(8글자)를 계산한다.
- 천간(天干) / 지지(地支) 조합
- 대운(大運) 흐름 계산 (10년 단위)
- 세운(歲運) 계산 (매년 운)

### 2단계 — 인생 이벤트 수집 (동적 데이터)

유저에게 시간축(타임라인) 형태로 이벤트를 입력받는다.

카테고리 5영역:
- 💰 재물 / 사업
- ❤️ 관계 / 애정
- 🏥 건강
- 🎓 명예 / 커리어
- 🏠 환경 (이사, 가족 등)

### 3단계 — LLM이 이벤트를 수치화

```json
{
  "event": "스타트업 창업, 투자 유치 성공",
  "date": "2018-03",
  "scores": {
    "재물":   0.75,
    "명예":   0.60,
    "관계":   0.10,
    "건강":  -0.05,
    "환경":   0.20
  },
  "intensity": 0.82,
  "valence": 0.65,
  "turning_point": true
}
```

수치화 기준:
| 값 범위 | 의미 |
|---|---|
| +0.8 ~ +1.0 | 매우 길한 사건 |
| +0.3 ~ +0.8 | 긍정적 사건 |
| -0.3 ~ +0.3 | 중립 |
| -0.8 ~ -0.3 | 부정적 사건 |
| -1.0 ~ -0.8 | 매우 흉한 사건 |

### 4단계 — 사주 보정 (핵심 엔진)

**기존 사주 예측값** vs **유저 실제 경험 이벤트 점수** 대조

- 어떤 십성(十星)에 민감하게 반응하는 사람인지 파악
- 길흉의 발현 시점 시차 보정
- 특정 오행(목화토금수)의 실제 영향력 가중치 개인화

### 5단계 — 개인화된 운세 해석 출력

**일반 서비스**: "2024년은 편재운으로 재물 기회가 있습니다"

**나의 사주**: "당신의 패턴 분석 결과, 편재운이 활성화될 때 실제 재물 변화가 발생할 확률이 높습니다. 특히 2020년처럼 관살이 겹치는 구간에서는 수익 기회가 와도 외부 압박으로 손실로 전환된 이력이 있습니다. 2024년 하반기는 유사한 구조이므로 리스크 관리를 먼저 고려하세요"

### 6단계 — 피드백 루프

```
이벤트 누적
    → 개인 사주 모델 정밀화
        → 해석 정확도 향상
            → 유저 Lock-in 강화
                → 이탈 방지
```

유저가 쓸수록 나만의 사주 모델이 완성되는 구조. 다른 서비스로 이전하면 축적된 데이터를 잃게 된다.

## 3. 이벤트 불일치 시나리오

불일치는 오류가 아니라 **가장 중요한 보정 데이터**다. 일치보다 더 많은 정보를 담고 있다.

### Type A — 방향 불일치 (예측 +, 실제 -)

```
사주 예측: 2019년 → "식신운, 재물과 건강 모두 길"
실제 이벤트: 재물 -0.7 (사업 실패)
```

**분석 프로세스:**
1. 이 유저의 사주 내 식신의 강도 확인
2. 세운과의 충(沖) 여부 확인
3. 결론: "이 유저는 식신운이 재물보다 건강에 더 발현되는 체질"
   - 재물 가중치: 식신 0.3 → 0.05로 하향
   - 건강 가중치: 식신 0.4 → 0.75로 상향

### Type B — 강도 불일치 (방향은 맞는데 임팩트가 다름)

```
사주 예측: 2022년 → "비견운, 소폭 손실 예상"
실제 이벤트: 재물 -0.95 (파산 수준의 손실)
```

**분석 프로세스:**
1. 과거 이력 전체 스캔 → 재물 이벤트가 항상 극단적으로 발현됨
2. 중간값(0.3~0.5대)이 없음 → "모 아니면 도 체질"
3. 결론: `intensity_multiplier = 1.5` 로 보정 저장

### Type C — 시점 불일치 (내용은 맞는데 시기가 다름)

```
사주 예측: 2021년 → "정관운, 승진/명예 상승"
실제 이벤트: 2023년 → 명예 +0.8 (승진)  ← 2년 늦게 발현
```

**분석 프로세스:**
1. 전체 이벤트에서 시점 패턴 추출
2. 이 유저의 "운 발현 시차" = 평균 +1.3년 늦게 반응
3. 결론: `lag_offset = +1.3년` 저장, 이후 모든 예측에 자동 적용

### 3회 이상 연속 불일치 시

"사주 해석 신뢰도 경고" 발동 → LLM이 유저에게 직접 질문
- 가족/파트너의 재물 이벤트가 대신 영향을 미친 경우
- 본인 명의가 아닌 다른 사람 명의로 활동한 시기
- 기억에 남지 않는 더 큰 이벤트가 있었는지

→ **누락된 이벤트를 역으로 발굴**

### 개인 사주 오차 프로파일 (Error Profile)

```json
{
  "재물_방향_정확도": 0.71,
  "명예_방향_정확도": 0.89,
  "관계_방향_정확도": 0.45,
  "평균_시차": "+1.3년",
  "강도_배율": 1.5,
  "최약_영역": "관계",
  "최강_영역": "명예"
}
```

→ 잘 맞는 영역은 더 구체적으로, 안 맞는 영역은 "변수가 많아 단정하기 어렵습니다"로 솔직하게 출력
→ **과신하지 않는다는 신뢰도 자체가 유저 충성도로 연결**

## 4. 출생 시각 역산 보정 시스템

### 핵심 개념 전환

```
기존 관점: "사주가 틀렸다" → 해석 모델 보정
새로운 관점: "입력값이 부정확하다" → 생년월일시 자체를 역산
```

인생 이벤트 데이터가 충분히 쌓이면, **실제 삶에 가장 잘 맞는 사주를 거꾸로 찾아낼 수 있다.**

### 왜 시각이 핵심인가

- 생년 → 거의 확실
- 생월 → 확실
- 생일 → 확실
- **생시 → 불확실** ← 여기가 변수
- 시주(時柱)는 2시간 단위로 바뀜 → 하루에 12개의 다른 사주가 존재

### 탐색 알고리즘

유저가 입력한 생시를 고정값이 아닌 **탐색 범위**로 처리

```
유저 입력: "오전 11시쯤 태어났어요"

탐색 범위:
  오전 9시  (사시 巳時)
  오전 11시 (오시 午時) ← 유저 주장
  오후 1시  (미시 未時)

→ 3개 사주를 동시에 생성
→ 각각의 대운/세운 흐름 계산
→ 유저 인생 이벤트와 대조
→ 가장 높은 일치율의 시주 채택
```

### 생일까지 범위 확장

- 시주 탐색 → 그래도 낮으면 → 생일 ±1일 범위
- 음력/양력 혼동 체크 (호적 정정 케이스)
- 제왕절개 예정일 vs 실제 분만 시각 구분

### 보정 우선순위

```
불일치 발생
    ↓
Step 1. 시주 후보 탐색 (±2시간 범위)
    ↓ 여전히 안 맞으면
Step 2. 음력/양력 변환 오류 체크
    ↓ 여전히 안 맞으면
Step 3. 생일 ±1일 탐색
    ↓ 여전히 안 맞으면
Step 4. 해석 모델 가중치 보정 (기존 방식)
    ↓ 여전히 안 맞으면
Step 5. "이 영역은 변수가 많습니다" 고지
```

**해석 모델을 건드리는 건 가장 마지막 수단**으로 밀어낸다.

이벤트 데이터가 30개 이상 쌓이면 **사실상 출생시각을 역산해내는 수준**까지 도달 가능. 이게 서비스의 진짜 킬러 기능.

## 5. 수익 모델

### 핵심 설계 원칙

> **핵심 변동비 = LLM API (사주 해석 output 토큰)**
> → 해석 횟수를 과금 기준으로 설계

```
일일 운세 (짧은 output)   → 건당 비용 낮음 → 구독으로 묶어서 박리다매
이벤트 보정 (중간 output) → 건당 비용 중간 → 크레딧으로 직접 연동
심층 분석 (긴 output)     → 건당 비용 높음 → 단건 고가 판매
```

비용과 수익이 정렬(align)된 구조 → 마진율이 티어별로 안정적 유지

### 비용 구조 (참고)

| 항목 | 수치 |
|---|---|
| 해석 1회 LLM 비용 | ≈ ₩20~40 (output 토큰 기준) |
| PLUS 마진 (30회 기준) | ≈ 70% |
| PRO 손익분기 | ≈ 330회/월 |

### 3가지 사용자 유형

**① 일상 소비형** — "오늘 운세 알려줘"
- 깊이보다 빈도가 중요, 카카오/네이버 운세처럼 쓰는 패턴
- LLM output이 짧고 단순 → 건당 비용 낮음
- **구독 모델** 적합

**② 정밀 탐구형** — "내 사주를 제대로 알고 싶어"
- 이벤트를 열심히 입력, 자신만의 정확한 사주 모델 완성 목표
- 서비스의 핵심 타깃, Lock-in이 가장 강하게 걸림
- LLM output이 길고 정교 → 건당 비용 높음
- **이벤트 입력 횟수 / 심층 분석 횟수 기반 크레딧** 적합

**③ 이벤트 드리븐형** — "중요한 결정 앞두고 사주 봐야겠어"
- 취업, 결혼, 이사, 창업 같은 인생 전환점에서만 접속
- 평소엔 조용하다가 특정 시점에 고관여 소비
- **단건 고가 결제** 적합

## 6. 커뮤니티 전략

### 왜 이 서비스에서 커뮤니티가 특별히 강력한가

유저들이 공유하는 팁이 곧 **서비스 정확도를 높이는 데이터 레이블링** 역할을 한다.

> "이혼을 -0.9로 입력했더니 보정이 이상해졌고, -0.3으로 넣었더니 훨씬 맞더라"

이런 팁 하나가 단순한 노하우가 아니라 **이벤트 수치화 방법론 자체를 커뮤니티가 집단지성으로 정교화하는 과정**이다.

### 예상되는 팁의 종류

**이벤트 입력 방법론**
- "실패한 사업을 하나의 이벤트로 넣는 것보다 준비기/절정기/마무리기 3개로 나눠 넣었더니 대운과 훨씬 잘 맞았다"
- "감정적 기억이 아닌 실질적 영향 기준으로 수치를 매겨야 함. 슬펐어도 결과적으로 좋은 일이면 플러스로"
- "가족의 큰 사건도 같이 입력하는 게 내 사주 보정에 영향을 줬다"

**출생 시각 역산 활용**
- "산모수첩을 찾아서 정확한 시각을 입력했더니 보정 정확도가 확 올라갔다"
- "제왕절개라 예정일과 실제 분만 시각이 달랐는데 어느 걸 써야 하나"

**해석 읽는 방법**
- "재물 영역은 잘 맞는데 관계 영역은 나한테 잘 안 맞더라. 영역별로 신뢰도를 다르게 봐야 한다"

→ **역술가가 수십 년 상담으로 쌓는 노하우를 유저 집단이 몇 년 만에 만들어내는 것**

### 커뮤니티가 만들어내는 비즈니스 확장

**파워유저 → 사주 코치 탄생**
- 팁을 많이 공유하고 검증된 유저 → 커뮤니티 내 신뢰 확보
- "사주 코치"로 공식화 → 다른 유저에게 이벤트 입력 컨설팅 유료 연결
- 플랫폼이 수수료를 가져가는 구조

**집단 검증 이벤트 수치 가이드**
- "결혼은 평균적으로 +0.6~0.8 사이가 적절하다"는 집단 합의 형성
- 신규 유저 온보딩 가이드라인으로 제공 → 초기 이탈률 감소

**커뮤니티 자체가 마케팅**
- "이렇게 입력했더니 10년 전 일이 정확히 맞았다"는 후기
- 광고보다 강한 바이럴 효과

### 커뮤니티 설계 주의점 — 개인 데이터 공개 범위

사주와 인생 이벤트는 매우 민감한 정보.

> **커뮤니티 공유 단위를 "이벤트 내용"이 아닌 "입력 방법론"으로만 제한**

- 구체적 내용은 익명화하거나 유저가 선택적으로 공개
- "내 대운 흐름"은 공유 가능, "2020년에 이혼했다"는 직접 노출 방지
- 이 구조가 신뢰를 쌓는 핵심

### 커뮤니티 플라이휠

```
유저가 많아짐
    → 팁이 쌓임
        → 신규 유저 진입 장벽 낮아짐
            → 유저가 더 늘어남
                → 경쟁사가 따라오기 어려워짐
```

## 7. 오프라인 역술가 연계 전략

### 역술가 입장에서 이 서비스가 필요한 이유

오프라인 역술가들의 가장 큰 고충: **기억과 고객 관리**
- 손님이 1년 후 다시 오면 저번에 무슨 얘기를 했는지 기억이 안 남
- 단골 손님임에도 매번 처음부터 다시 설명해야 함

이 서비스가 역술가에게 제공하는 것:
- **고객 관리 시스템** + **사주 계산 도구** + **상담 히스토리 관리**

### B2B2C 구조

```
역술가가 서비스 도입
       ↓
상담 전 손님에게 앱 설치 권유
"미리 이벤트 입력해오시면 더 정확하게 봐드립니다"
       ↓
손님이 앱에서 인생 이벤트 입력
       ↓
역술가는 태블릿으로 보정된 사주 데이터 확인하며 상담
       ↓
상담 후 손님 앱에 히스토리 저장
       ↓
손님은 일상에서 앱 계속 사용 → 서비스 Lock-in
```

**역술가가 유저 획득 채널이 되는 구조**. 마케팅 비용 없이 오프라인 고객이 그대로 앱 유저로 전환.

### 역술가에게 줄 수 있는 것

**① 상담 전 데이터 정리**
- 손님 인생 이벤트 타임라인이 역술가 화면에 미리 정리
- 30분 상담 시간을 "파악"이 아닌 **"해석"에만 집중** 가능
- 상담 질 향상, 손님 만족도 향상

**② 출생 시각 역산 보조**
- 역술가가 감으로 물어보던 것을 → 데이터 기반으로 제시
- "입력하신 이벤트들을 보면 오후 1시 생이 훨씬 잘 맞습니다"
- 역술가의 전문성이 데이터로 보강

**③ 상담 이력 관리**
- 손님별 히스토리 누적
- "작년에 제가 뭐라고 했죠?" 확인 가능
- 단골 관리 체계화

### 핵심 포지셔닝 원칙

> "AI가 사주를 본다"가 아니라  
> **"역술가의 능력을 더 잘 발휘하게 해주는 도구"**

역술가의 전문성을 위협하는 것이 아니라 보조하는 것으로 일관되게 포지셔닝.

### 초기 진입 전략

역술가들의 네트워크(명리학 학회, 스터디 모임, 온라인 카페)를 활용.
- **유명 역술가 몇 명을 먼저 공략** → 오피니언 리더 확보
- 유명 역술가가 쓴다는 것 자체가 일반 유저에게 신뢰 신호

**역술가 1명의 LTV = 일반 유저 수백 명의 가치**

## 8. 멀티테넌트 플랫폼 전략

### 개념

각 역술가가 단순히 "이 앱을 쓰는 사람"이 아니라 **자신만의 브랜드 공간을 가진 사업자**가 되는 구조.

```
naui-saju.com/혜담역술원
naui-saju.com/김철수사주
naui-saju.com/운명연구소
```

### 테넌트별 커스터마이징 범위

| 레이어 | 커스터마이징 가능 항목 |
|---|---|
| 브랜딩 | 상호명, 로고, 색상, 소개, 전문 분야, 후기 |
| 상담 운영 | 메뉴 구성, 가격 설정, 예약 시간, 비대면·대면 선택, 사전 질문지 |
| 고객 관리 | 고객 DB, 상담 히스토리, 메모, 알림, 생일 메시지 |
| 데이터 | 내 고객 사주 통계, 내 예측 적중률 트래킹 |

### 공유 vs 분리

| 공유 (플랫폼 자산) | 분리 (테넌트 자산) |
|---|---|
| 사주 계산 엔진 | 고객 DB |
| LLM 해석 모델 | 상담 히스토리 |
| 이벤트 수치화 로직 | 역술가 브랜딩 |
| 빅데이터 학습 결과 | 가격 정책 |
| 커뮤니티 (선택적) | 예약 캘린더 |

**고객 데이터는 테넌트 소유**로 명확히 보장.
단, 익명화된 통계 데이터는 플랫폼이 학습에 활용 (동의 기반).

### 수요 검증 먼저, 개발은 나중

멀티테넌트는 기술적으로 큰 작업 → 수요 검증 없이 만들면 오버엔지니어링.

**페이즈 1에서 먼저 수요를 검증하는 방법:**
1. 역술가 몇 명에게 일반 계정 제공 → 반응 관찰
2. "역술가 파트너 신청" 폼만 공개 → 신청 수가 50명 이상이면 개발 착수 신호
3. 카카오채널/네이버 예약으로 임시 연동 → 정식 기능 없이 수요 테스트

**개발 착수 판단 기준:**
- 역술가 파트너 신청이 50명 이상 들어올 때
- 기존 역술가들이 자발적으로 추천하고 다니기 시작할 때
- 일반 유저가 "제가 다니는 역술원에도 써달라"는 요청이 반복적으로 올 때

> **만들고 검증하는 게 아니라 검증하고 만드는 순서**

### 최종 비전

```
초기  →  사주 앱
중기  →  역술가 SaaS 도구
후기  →  명리학 업계 인프라 플랫폼
최종  →  인류 인생 패턴 데이터 플랫폼
```

**명리학 업계의 Shopify.** 역술가들이 이 플랫폼 없이는 영업을 못 하는 수준이 되면 압도적인 해자(moat) 완성.

## 9. 확장 전략 — 국내 → 해외 → 글로벌

### 1단계 — 국내 런칭 (데이터 수집 기간)

**목표: 성장보다 데이터 품질 확보**

- 한국 유저가 가장 솔직하고 풍부하게 인생 데이터를 입력
- 이 시기에 집중해야 할 것: 연령대별, 성별, 직업군별로 어떤 이벤트 유형이 어떤 사주 패턴과 높은 상관관계를 보이는지 축적
- 만들어야 할 것: **"이벤트 수치 — 실제 대운 일치율" 데이터셋**

### 2단계 — 해외 런칭

**일본 1순위인 이유**
- 사주(四柱推命)가 이미 체계화돼 있음
- 유료 결제에 익숙
- 개인 데이터를 정교하게 입력하는 성향이 강함
- 이 서비스의 UX("과거 이벤트를 꼼꼼히 입력")에 가장 잘 맞는 나라

**대만·홍콩**
- 한자 문화권 → 사주 팔자 개념 그대로 가져갈 수 있음
- 번체자 대응만으로 진입 가능

**동남아(베트남, 태국)**
- 점성술 문화가 강하지만 사주 체계가 다름
- 개념의 번역이 아니라 **재설계** 필요

**해외 진출의 핵심 기술 과제**
> 나라마다 "인생 이벤트의 의미와 무게"가 다름

- 한국: 수능 실패 → -0.8
- 일본: 재수가 일반적 → -0.4
- 이혼에 대한 사회적 낙인, 직업 이직의 빈도와 의미도 다름

→ **이벤트 수치화 모델을 국가별로 캘리브레이션** 해야 함

### 3단계 — 글로벌 통합

- 동남아·서양권 확장
- 타 점성술 체계와의 병렬 데이터 수집
- 글로벌 인생 패턴 데이터셋 완성

## 10. K-Culture 서양권 진입 전략

### 왜 K-Culture 프레임이 유효한가

서양권에 사주를 "동양 신비주의"로 팔면 진입장벽이 높다. K-Culture 프레임은 완전히 다르다.

- 한국 여행을 가서 직접 점을 보고 온 서양인들이 이미 있음
- 그들에게 사주는 "낯선 것"이 아니라 **"한국에서 경험한 인상적인 문화 체험"**
- 이미 감정적 연결고리가 있는 상태에서 접근

### 구체적 진입 전략

**"한국에서 직접 경험한 그 점집, 이제 앱으로"**

> *"You got your saju read in Seoul. Now continue the journey."*

한국 여행 중 경험을 앱으로 확장하는 내러티브. TikTok, YouTube, Instagram에서 자연스럽게 닿을 수 있음.

**K-드라마 팬 커뮤니티 연결**
- K-드라마에서 사주 장면 등장 → 팬들의 "저게 뭔지 궁금했다" 반응 공략
- 드라마 속 사주 장면을 영어로 설명하는 콘텐츠 → 서비스로 연결

**K-팝 아이돌 사주 콘텐츠**
- 공식 생년월일이 공개된 아이돌 사주 분석
- 팬덤 안에서 서브컬처로 이미 존재
- 서비스의 입문 콘텐츠로 활용

### 서양 점성술 유저 공략

> "Western astrology vs Korean Saju — what's different?"

- 서양권 점성술 유저 (이미 수억 명)를 자연스럽게 끌어들이는 입구
- 사주를 설명하는 비용 없이 비교 콘텐츠로 흥미에서 몰입으로

### 역방향 관광 연계

서양권에서 앱을 쓰다가 "한국에 가서 직접 점을 보고 싶다"는 니즈 발생
→ 한국 전통 점집, 사주 카페와 연결하는 **역방향 관광 상품**
→ "디지털 사주 + 오프라인 한국 체험" 패키지
→ 여행사와 협업, K-관광 콘텐츠로서의 가치

## 11. 빅데이터 기반 新 명리학

### 왜 이것이 전례 없는 일인가

명리학의 수천 년 한계: **샘플이 없었다.**
역술가 개인의 기억과 경험에만 의존했고, "이 사주를 가진 사람들의 실제 인생"을 체계적으로 모은 데이터가 존재하지 않았다.

이 서비스가 수년간 운영되면 **처음으로 그 데이터가 생긴다.**

```
특정 사주 구조를 가진 사람 10만 명의
실제 인생 이벤트 타임라인 데이터
```

### 데이터로 할 수 있는 것

**기존 명리학 검증**
- "이 사주는 재물운이 강하다" → 실제로 몇 %가 재물 이벤트 +0.7 이상을 경험했나?
- 처음으로 명리학이 통계적 검증을 받는 순간

**새로운 패턴 발견**
- 기존 명리학이 설명하지 못하는 패턴이 데이터에서 출현 가능
- "이 십성 조합을 가진 사람들은 40대가 아니라 35~37세에 재물 전환점이 오는 경향"
- 이를 역으로 명리학 이론에 피드백 → **데이터로 갱신되는 살아있는 명리학**

### 글로벌 통합 가설

> 동양권 사주, 서양 점성술, 인도 베딕 점성술은 서로 다른 체계처럼 보이지만,  
> 실제로 인간의 인생 패턴을 설명하는 **공통된 무언가**를 각자의 방식으로 표현하고 있는 건 아닐까?

- 다양한 문화권의 인생 데이터가 쌓이면 사주와 베딕 점성술이 동일 인물의 인생을 얼마나 다르게/비슷하게 예측하는지 실증적 비교 가능
- 문화권이 달라도 특정 주기(29~30세 토성 회귀, 대운 전환기)에서 인생 이벤트 발생 빈도가 공통적으로 높다면 → **학술적으로도 전례 없는 발견**

### 최종 포지셔닝 전환

```
Phase 1~3: 사주 앱
Phase 4:   "인간 삶의 패턴을 데이터로 이해하려는 프로젝트"
```

이 시점에서 투자 유치, 학술 협력, 미디어 노출이 완전히 다른 레벨에서 이뤄진다.

──────────────────────────────
· 데이터 기반 명리학 이론 갱신

## 핵심 경쟁 우위 요약

|---|---|---|
